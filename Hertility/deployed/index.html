Data Streams

Hertility collects data from three platforms — the website, the Android app, and the iPhone app. Each maps to a Google Analytics data stream, which means you can report on each platform separately or look at the aggregate.

When building reports in GA, it usually makes sense to filter by stream name so you're looking at the right platform's data.

02

Google Tag Manager

Google Tag Manager (GTM) is the central routing layer in the data pipeline. When a user takes an action on the website or app, GTM picks up the event and sends it to the right marketing platforms.

GTM currently routes data to Google Analytics, Google Ads, Meta (Facebook/Instagram), LinkedIn, Pinterest, Awin, and others. Rather than hard-coding tracking on each platform individually, GTM lets the team manage all tags from one place.

The full list of active tags and their status is maintained in the Marketing Tags Tracker.

03

Google Analytics & BigQuery

Google Analytics (GA) is the primary reporting tool. It powers key business reports including the "Infinity Loop" report and other business-critical dashboards.

GA data is exported daily to BigQuery — Google's data warehouse. The Engineering team writes SQL queries to model this raw data for upstream reporting and analysis.

Sometimes the GA user interface can't answer a specific question. When that happens, the raw BigQuery data is the go-to resource — it contains everything GA captures, in a queryable format.

04

Engineering & the Data Layer

The data layer is a JavaScript object (dataLayer) that lives on the website. Engineering implements data layer pushes to capture specific user interactions — things like video engagements, button clicks, and form submissions.

When Engineering adds a new dataLayer.push() event, GTM can be instrumented to listen for it and routes the data to the appropriate marketing platforms.

This keeps tracking clean and maintainable — the website code only needs to "announce" what happened, and GTM handles the rest.

Example

// When a user clicks a CTA button, Engineering pushes
// an event to the data layer:

dataLayer.push({
  event: 'cta_click',
  button_text: 'Book a Test',
  page_section: 'hero'
});

// GTM picks this up and sends it to GA, Google Ads,
// Meta, etc. — no extra code needed on the website.
05

Server-side Tracking & Stape

In a traditional setup, marketing data is sent directly from the user's browser to each platform — Google Analytics, Meta, Google Ads, and so on. This is called client-side tracking, and it's how most websites work. The problem is that browsers are getting stricter. Ad blockers, iOS privacy changes, and shorter cookie lifespans all chip away at the data that actually makes it through.

Server-side tracking adds a middle step. Instead of the browser talking directly to every platform, it sends data to a server that Hertility controls first. That server then forwards the data onward to Google Analytics, Google Ads, Meta, and the rest. Because the data passes through Hertility's own infrastructure, it's less likely to be blocked or lost along the way.

In the context of GTM, client-side tracking is handled by your GTM web container, whilst server-side tracking is handled by your GTM server container.

Hertility runs its server-side container through a service called Stape. Stape hosts and manages the server-side GTM environment so that Hertility's Engineering team doesn't need to maintain the servers themselves.

In practice, the flow looks like this: a user takes an action on the website → the browser-side GTM container picks it up → instead of sending data straight to Meta or Google, it sends it to the Stape-hosted server → the server processes and forwards it to the final platforms.

Day-to-day, you don't need to interact with Stape directly — it runs in the background. But it's worth knowing it exists. It's also worth knowing that there are two GTM containers — a client-side container that runs in the user's browser and a server-side container hosted by Stape that receives data from the web container and forwards it on to the platforms.

Website Platform Complexity

07

Cookie Consent

No marketing data fires unless the user consents. The CookieBot (UserCentrics) cookie banner appears when a user first visits, and their choice determines what data is collected.

If a user declines cookies, marketing tags do not fire — no data is sent to Google Analytics, Google Ads, Meta, or any other platform. Only users who consent are tracked.

Cookie consent is managed by GTM, which effectively 'links' user consent preferences to the marketing triggers.This is the very first gate in the data pipeline: before anything else happens, consent must be given.
